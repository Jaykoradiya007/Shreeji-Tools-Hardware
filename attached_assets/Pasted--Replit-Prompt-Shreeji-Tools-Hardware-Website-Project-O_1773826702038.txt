## Design Direction

**Style**: Dual-tone, minimal, clean, and industrial. The design should feel crafted specifically for the tools & hardware industry — not generic.

**Design Inspiration Keywords**: Industrial precision · Machined metal · Blueprint grids · Workshop aesthetic · Utilitarian elegance

**Typography**:
- Use **Google Fonts** only (free CDN).
- Display/Heading font: Something bold and industrial — e.g. `Barlow Condensed`, `Rajdhani`, or `Bebas Neue`.
- Body font: Clean and legible — e.g. `DM Sans`, `Nunito Sans`, or `Outfit`.
- Avoid Inter, Roboto, Arial, and generic system fonts.

**Color Scheme — Dual Tone (Light Base)**:
- Primary/Background color: White (`#FFFFFF`) or Off-White (`#F7F6F2`) — used for page background, section backgrounds, and cards.
- Secondary/Accent color: Pick ONE bold accent that contrasts beautifully against white and feels industrial — best options:
  - Safety Orange (`#E8570A`) — energetic, workshop-grade, high visibility
  - Steel Blue (`#1A3C5E`) — professional, trusted, corporate strength
  - Amber Gold (`#C8860A`) — premium, warm, established business feel
  - Signal Red (`#C1121F`) — bold, powerful, attention-grabbing
- The accent color is used for: headings, CTA buttons, card borders on hover, icon fills, badges, section dividers, and nav highlights.
- Use CSS custom properties (variables) for all colors — defined once at `:root`.
- Background must feel clean, airy, and professional — white/off-white base with the accent color as the hero tone.
- Sections can alternate between pure white (`#FFFFFF`) and soft off-white (`#F7F6F2`) for visual separation — no dark backgrounds anywhere on the page except optionally the very top announcement bar and footer.

**Layout & Feel**:
- Asymmetric, bold section headers with large typography.
- Use geometric accent shapes (thin lines, corner brackets, diagonal dividers) in the accent color as decorative elements — no clipart.
- Generous whitespace throughout — the white base needs breathing room to feel premium, not empty.
- Smooth scroll behaviour across the whole page.
- Sticky navigation bar — white background with a subtle bottom border, gains a soft box-shadow on scroll.
- Subtle entrance animations on scroll using CSS `@keyframes` + `IntersectionObserver` — staggered reveals for cards and sections.
- Hover effects on product category cards (accent color border highlight, subtle upward lift — no heavy shadows).
- Text colors: near-black (`#1A1A1A`) for headings, dark grey (`#444444`) for body text — never pure black on white.

## GitHub Pages Development Guidelines
### Platform constraints — build accordingly
- This website will be hosted on GitHub Pages. The developer must treat it as a
  purely static site. No server-side code of any kind — no PHP, Node.js, Python,
  Ruby, or any backend language. Everything must run entirely in the browser.
### What NOT to do
- Do NOT use server-side form handling. All contact/enquiry forms must use a
  third-party service such as Formspree or Web3Forms for form submission.
- Do NOT store any API keys, passwords, or secret tokens in JavaScript files.
  GitHub Pages repos are public — any key in the code is visible to everyone.
- Do NOT upload videos directly to the repository. Embed videos via YouTube or
  Vimeo iframe instead.
- Do NOT use .htaccess files, server redirects, or any Nginx/Apache config.
  These have no effect on GitHub Pages.
- Do NOT build a backend-dependent feature (user login, database calls, dynamic
  inventory) and expect it to work — it will not.
- Do NOT commit large uncompressed images. Each file must stay under 100 MB and
  the total repository must stay under 1 GB.
### What TO do — GitHub Pages friendly practices
- Build with plain HTML, CSS, and vanilla JavaScript (or a static site generator
  like Jekyll or Hugo if needed).
- Compress all images before committing. Use WebP format wherever possible.
- Use a free CDN for fonts (Google Fonts) and icons (Font Awesome CDN) — do not
  bundle large font files in the repo.
- Use Formspree or Web3Forms for the contact form — these are free and require
  only a small HTML snippet, no backend.
- For any third-party integrations (maps, analytics, chat widgets), use their
  official embed/iframe snippet only — no server calls.
- Keep the total repository size well under 500 MB as a safe buffer.
- Minify CSS and JS files before the final deployment push.
- Test all pages as static files locally before pushing — open index.html
  directly in a browser with no local server running.
### Deployment reminder
- The site deploys automatically from the main branch. Every push triggers a
  rebuild. Batch all changes into single commits to avoid hitting the
  10 builds/hour rate limit.
- The live URL will be: https://[username].github.io/[repo-name]
  Replace with the actual repo name once confirmed.
